¿Qué es el Responsive Web Design?

RWD: Es el nombre que damos al diseño/maquetación adaptable

  • Ethan Marcotte: ese tipo tan práctico que escribía en A List Apart
  • No es una tecnología. Es una técnica.
  • Está vinculada conceptualmente al front-end development
  • Trata de evitar que limitemos nuestra visión de la web (folio)

Las tres partes del RWD

RWD: Un diseño adaptable tiene tres patas

  • Un grid flexible
  • Imágenes y medios flexibles
  • Media queries

Grid flexible

RWD: Los grids, muy empleados en diseño y maquetación han evolucionado para facilitarnos la creación de páginas web

El grid más popular es el grid 960.

Este grid representa a la perfección el concepto de lienzo, con sus pros y sus contras.

La evolución natural de este modelo es el diseño flexible.

Para obtener un diseño flexible a partir de un grid 960

  • Cambiamos px por %
  • Puntos y píxeles pasan a ser em (1em ~= 16px)
  • Objetivo / contexto = resultado

Medios flexibles

Los medios son una parte importantísima en las webs modernas.

Si aspiramos a hacer una web adaptable nuestros medios tienen también que ser adaptables

Por suerte hay una regla que nos permite gobernar a todos los medios

img,
embed,
object,
video {
    max-width: 100%;
}

Pero no solo de max-width viven las imágenes, el overflow puede ser también nuestro aliado

Solo tienes que cambiar esto

.feature img {
    max-width: 100%;
}

Por esto

.feature {
    overflow: hidden;
}
.feature img {
    display: block;
    max-width: auto;
}

Las imágenes en todo caso suelen ser bastante "pacíficas". Hay enemigos peores.

En el caso de los vídeos, esta regla es eficaz... pero puede no ser suficiente.

Por suerte tiene remedio con un poco de html y css

.video_sample{
    height: 0;
    padding-bottom: 56.25%;
    position: relative;
    width: 100%;
}
                    
.video_sample object,
.video_sample iframe {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
}

Cuando trabajamos con fondos un poco de css3 también puede ser de ayuda

img{
    background-image: url(./path/to/img.jpg);
    background-size: 100%;
    background-repeat:no-repeat;
}

Media Queries

Hasta aqui hemos avanzado mucho, pero nuestra web sigue adoleciendo de muchos problemas.

La solución a nuestros problemas se llama "media query".

Permite cambiar la hoja de estilos en función de información sensible a las características de la pantalla desde la que estamos viendo la web.

Un ejemplo sencillo de media query es el siguiente:

@media screen and (min-width: 1024px) {
    body {
        font-size: 100%;
    }
}

Podemos incluir las media queries de infinidad de formas diferentes. Como una etiqueta link.

link rel="stylesheet" href="wide.css" media="screen and (min-width: 1024px)"

Un import

@import url("wide.css") screen and (min-width: 1024px);

O directamente dentro de una hoja de estilos ya existente

@media only screen and (max-device-width: 480px) {
    ...
}

Un ejemplo de una query completa podría ser el siguiente:

/* iPads (landscape) */
@media only screen 
    and (min-device-width : 768px) 
    and (max-device-width : 1024px) 
    and (orientation : landscape) {
    /* Styles */
}

A fin de ser completos. Lista de parámetros que puedes consultar en una media query

width
Ancho del display.
height
Alto del display.
device-width
Ancho del área de renderizado del dispositivo.
device-height
Alto del área de renderizado del dispositivo.
orientation
portrait o landscape
aspect-ratio
Relación de aspecto de la ventana del navegador (p.ej.)
device-aspect-ratio
Relación de aspecto del dispositivo.
color
Número de bits de color de un dispositivo. Un dispositivo de 8 bits devolvería 8, uno sin color 0.
color-index
Número de entradas de color en la tabla de lookup del dispositivo de salida. Por ejemplo @media screen and (min-color-index: 256)
monochrome
Nos permite saber el nº bits por pixel en un dispositivo monocromo.
resolution
ensidad de píxeles en el dispositivo: screen and (resolution: 72dpi) or screen and (max-resolution: 300dpi).
scan
En televisiones nos permite saber si el procesado es progressive o scan
grid
Comprueba si el dispositivo tiene un display basado en grid. Como los teléfonos con un único tamaño de fuente.

Consejos generales

No hay una regla exacta para definir cuándo tu web tiene que "saltar"... Pero conocer media queries de los distintos dispositivos puede ayudar.

Trata de ser límpio con tus CSS y tu HTML. Lo primero es ser semánticamente correcto.

Cuidado con las CSS de la muerte, si las media queries se solapan puede ser un caos (reitero, hay que ser ordenado).

En la medida de lo posible trabaja mediante "progressive enhancement".

El RWD es una tecnología de front end. No te embarques en mortíferas batallas de backend para resolver un problema que no es de backend.

Imágenes mucho más que flexibles

Las imágenes, pese a ser sencillas de lidiar, son el principal problema del RWD.

Hasta que "picture" no sea una realidad tenemos un serio problema con los tamaños de las imágenes.

<picture alt="Angry pirate">
    <source src="hires.png" media="min-width:800px">
    <source src="midres.png" media="min-width:480px">
    <source src="lores.png">
    <!-- fallback for browsers without support -->
    <img src="midres.png" alt="Angry pirate">
</picture>

Llegados a este punto hay dos soluciones:

  • Sentase y esperar
  • Recurrir a JS para atacar el problema

Si recurrimos a JS tenemos que tener claro el problema a solucionar, incluir más de un src a una misma imagen.

La mejor aproximación es la siguiente:

<img src="t.gif" data-src="real-image.jpg" data-bigger-src="real-bigger-image.jpg">

Si os interesa esta línea de trabajo mi aproximación preferida es la de Jake Archibald
